üìö Understanding Plugin Architecture

What is a Plugin?

A plugin in Editora is a modular component that extends the editor's functionality. Plugins can:

  • Add new toolbar buttons and commands
  • Create custom node types in the document
  • Handle keyboard shortcuts
  • Provide UI components and interactions
  • Integrate with external APIs
  • Transform and validate content

Core Plugin Concepts

üîë Key Concepts:
  • ProseMirror: Editora is built on ProseMirror, a powerful editor framework
  • Schema: Defines the structure of editable content (nodes and marks)
  • Commands: Functions that modify editor state
  • Plugins: Extend editor behavior with new features
  • Providers: React components that manage plugin state and UI

Plugin Anatomy

Every Editora plugin consists of three main parts:

Component Purpose Example
Node/Mark Definition Defines how content is stored in the document MyCustomNode.ts
Commands Functions that insert/modify the custom element insertMyNodeCommand
UI Provider React component for UI and state management MyNodeProvider.tsx

üöÄ Creating Your First Plugin - Callout Box

Let's build a practical plugin that adds a Callout Box (alert/note box) to the editor. This plugin will help users create highlighted information boxes.

1Define the Node (MyCustomPlugin.ts)

Start by defining the node structure and commands:

src/CalloutPlugin.ts
import { Node } from 'prosemirror-model'; import { Command } from 'prosemirror-state'; import { EditorView } from 'prosemirror-view'; // Define the Callout node export const CalloutNode = { name: 'callout', group: 'block', atom: true, // Define what data this node can store attrs: { type: { default: 'info' }, // info, warning, success, error message: { default: '' } }, // HTML representation toDOM(node: Node) { return [ 'div', { class: `callout callout-${node.attrs.type}`, data-type: node.attrs.type }, ['p', node.attrs.message] ]; }, // Parse HTML back to node parseDOM: [ { tag: 'div.callout', getAttrs(dom: HTMLElement) { return { type: dom.getAttribute('data-type') || 'info', message: dom.textContent || '' }; } } ] }; // Command to insert a callout export const insertCalloutCommand = ( type: 'info' | 'warning' | 'success' | 'error', message: string ): Command => { return ({ tr, dispatch }) => { const { $from } = tr.selection; const node = tr.doc.type.schema.nodes.callout.create({ type, message }); if (dispatch) { dispatch(tr.replaceRangeWith($from.pos, $from.pos, node)); } return true; }; };

2Create React Provider (CalloutProvider.tsx)

Now create the UI component that manages the callout interface:

src/CalloutProvider.tsx
import React, { useState } from 'react'; import { useEditorContext } from '@editora/core'; import { insertCalloutCommand } from './CalloutPlugin'; import './Callout.css'; export const CalloutProvider = () => { const { view } = useEditorContext(); const [message, setMessage] = useState(''); const [calloutType, setCalloutType] = useState'info'); const handleInsertCallout = () => { if (!message.trim()) { alert('Please enter a message'); return; } const command = insertCalloutCommand(calloutType, message); if (view && command(view.state, view.dispatch)) { // Reset form setMessage(''); setCalloutType('info'); view.focus(); } }; return ( <> {/* Toolbar Button - Add to editor toolbar */} <button onClick={() => {/* Open modal/form */}} title="Insert Callout Box" className="callout-button" > üí¨ Callout </button> {/* Modal/Form */} <div className="callout-form"> <div> <label>Callout Type:</label> <select value={calloutType} onChange={(e) => setCalloutType(e.target.value)} > <option value="info">‚ÑπÔ∏è Info</option> <option value="warning">‚ö†Ô∏è Warning</option> <option value="success">‚úì Success</option> <option value="error">‚úï Error</option> </select> </div> <div> <textarea value={message} onChange={(e) => setMessage(e.target.value)} placeholder="Enter your message..." rows={3} /> </div> <button onClick={handleInsertCallout}> Insert Callout </button> </div> </> ); };

üéØ Plugin Types & Examples

Block Plugins

Block-level elements that take up full width (examples: callout, quote, code block)

Use when: Creating elements that should be standalone blocks in the document

Inline Plugins

Elements that flow within text (examples: link, mention, emoji)

Use when: Creating elements that should appear within paragraphs or text

Mark Plugins

Text formatting marks (examples: bold, italic, highlight)

Use when: Creating text styling or formatting options

Command-Only Plugins

Plugins that provide functionality without creating new content types (examples: find/replace, undo/redo)

Use when: Creating utility features without new content types

Plugin Type Built-in Examples Idea for Custom Plugin

‚ú® Best Practices & Tips

Code Quality

  • Use TypeScript for type safety
  • Write unit tests for your plugin
  • Follow the existing code style
  • Document your code with JSDoc comments
  • Handle edge cases and errors gracefully

Performance

  • Memoize React components with React.memo
  • Use useCallback for event handlers
  • Avoid unnecessary re-renders
  • Lazy load heavy dependencies
  • Minimize CSS selectors complexity

User Experience

  • Provide clear visual feedback
  • Add keyboard shortcuts
  • Include helpful tooltips
  • Support undo/redo properly
  • Mobile-friendly design

Debugging Tips

‚ö†Ô∏è Common Issues & Solutions:
  • Node not appearing: Check that node is registered in schema
  • Command not working: Verify dispatch is being called correctly
  • Styles not applied: Ensure CSS is imported and class names match
  • Memory leaks: Clean up event listeners in useEffect cleanup

Testing Your Plugin

Basic Plugin Test
import { describe, it, expect } from 'vitest'; import { CalloutNode, insertCalloutCommand } from './CalloutPlugin'; describe('Callout Plugin', () => { it('should insert callout node', () => { const command = insertCalloutCommand('info', 'Test message'); expect(command).toBeDefined(); }); it('should have correct schema', () => { expect(CalloutNode.name).toBe('callout'); expect(CalloutNode.group).toBe('block'); }); });

Common Mistakes to Avoid
